# EutherHost Security Plan

This is the deferred hardening track for exposing EutherHost beyond a trusted LAN.

## Current Scope

- Treat the current host server as LAN/prototype only.
- Keep `.euther-host/` private and out of git.
- ROMs are exposed only through the configured ROM root and canonicalized relative paths.
- Login uses Argon2 password hashes and an HttpOnly session cookie.
- Login failures are rate-limited per IP and username in memory.
- Session timeout and Secure-cookie mode are configurable in `.euther-host/config.toml`.
- Login attempts and ROM launches are written to `.euther-host/audit.log`.
- Mutating authenticated routes require a per-session CSRF token.
- Host stream routes reject mismatched request origins when an `Origin` header is present.
- Host users have explicit play, ROM launch, ROM upload, and library-management permissions.
- Library writes can be locked down with `library_read_only = true`.
- Static files and ROM library paths are canonicalized and contained within their configured roots.
- Public deployment templates are available in `deploy/`, and `scripts/host-public-config.sh` writes a locked HTTPS config.

## Before Internet Exposure

1. Use `https://apothicthech.se` as the public origin.
2. Copy or install the local `deploy/Caddyfile` and proxy HTTPS to `127.0.0.1:32162`.
3. Keep `.euther-host/config.toml` generated by `scripts/host-public-config.sh https://apothicthech.se /home/nichlas/roms`.
4. Keep login rate limiting enabled per IP and per username.
5. Keep CSRF protection enabled for mutating HTTP routes.
6. Keep checking `Origin` on stream routes, and repeat that for future WebSocket routes.
7. Tune session expiry settings in `.euther-host/config.toml`.
8. Move sessions from memory to a persisted or restart-aware store if needed.
9. Review audit logging for login success/failure and ROM launches.
10. Keep read-only library mode enabled unless actively administering the ROM library.
11. Re-review all file-serving paths whenever new file endpoints are added.

## Safer Interim Option

Use Tailscale/WireGuard for remote access before opening ports to the public internet.

Recommended path:

1. LAN host mode.
2. Tailscale-only remote play.
3. HTTPS reverse proxy.
4. Public internet only after rate limiting, CSRF, origin checks, and logging are in place.

## Dedicated Server Plan

### Server Choice

- Prefer a small dedicated server or VPS with reliable single-core performance over lots of idle cores.
- Minimum starting point: 2 vCPU, 4 GB RAM, 40 GB SSD, unmetered or generous bandwidth.
- Choose a provider with simple DNS, reverse DNS, firewall controls, snapshots, and a clean Debian or NixOS install image.
- Keep ROM storage separate from the git checkout if the server provider offers an attached volume.

### Base OS

Debian path:

1. Install Debian stable with only SSH enabled.
2. Create a non-root deploy user, disable password SSH login, and use SSH keys only.
3. Enable unattended security upgrades.
4. Install `ufw`, allow only SSH, HTTP, and HTTPS.
5. Install Rust, Node, npm, Caddy, git, and build essentials.

NixOS path:

1. Install NixOS stable.
2. Define users, SSH keys, firewall ports, Caddy, and the EutherHost service declaratively.
3. Keep secrets and `.euther-host/` outside git.
4. Use a persistent data directory for `.euther-host/`, `.euther-bridge/`, logs, and ROMs.

### First Deploy

1. Point DNS `A`/`AAAA` records for `apothicthech.se` to the server.
2. Clone this repo on the server.
3. Build release artifacts with `scripts/build-release.sh` and `npm run build`.
4. Run `scripts/host-init-user.sh` once, then `scripts/host-public-config.sh https://apothicthech.se /path/to/roms`.
5. Install or adapt `deploy/Caddyfile` for Caddy.
6. Install or adapt `deploy/eutherhost.service.example` for systemd.
7. Start EutherHost on `127.0.0.1:32162`; let Caddy own public `80`/`443`.
8. Verify that `.euther-host/`, local Caddyfile, ROMs, saves, and logs are not tracked by git.

### Smoke Test

1. Visit `https://apothicthech.se` from a non-LAN connection.
2. Log in and confirm the session cookie has `Secure`, `HttpOnly`, and `SameSite=Lax`.
3. Start EutherDogs and confirm input, audio, and highscore writes.
4. Switch to Mega Drive manually and launch a ROM with an admin account.
5. Confirm a non-admin user can play but cannot upload ROMs or modify the library.
6. Check `.euther-host/audit.log` for login and ROM launch entries.

## Performance Track Notes

Phone playback is currently expected to be slow because the host stream is still the dev-style
frame/audio packet path. The next performance work should measure and reduce:

- frame payload size
- frame encode/decode cost
- HTTP request cadence
- shader/render cost on mobile
- audio scheduling overhead

Likely next steps are a persistent binary stream, frame delta/compression, or a real video/WebRTC
transport once the baseline bottleneck is measured.
